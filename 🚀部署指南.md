## 🔍 部署前检查

在开始部署前，请确认：

1. **网络连接** - 确保可以访问服务器
2. **SSH访问** - 确认SSH密钥或密码正确
3. **本地环境** - Node.js和npm已安装
4. **磁盘空间** - 服务器有足够的存储空间

## 📊 部署流程

### 阶段1：本地准备
1. 🔨 **构建前端** - 清理并重新构建生产版本
2. 📦 **创建包** - 打包所有必要文件
3. 🗜️ **压缩** - 创建部署压缩包
4. ⬆️ **上传** - 传输到服务器

### 阶段2：服务器部署
1. 🛑 **停止服务** - 安全停止所有相关服务
2. 💾 **备份数据** - 自动备份现有数据库
3. 🧹 **清空目录** - 完全清理旧文件
4. 📁 **创建结构** - 重建目录结构
5. 📤 **解压文件** - 部署新代码
6. 🔄 **恢复数据** - 恢复数据库（可选）
7. 📥 **安装依赖** - 安装Node.js包
8. ⚙️ **更新配置** - 更新Nginx等配置
9. 🚀 **启动服务** - 启动新应用

## 🚨 故障排除

### 常见问题及解决方案

#### 1. 前端构建失败
```bash
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run build
```

#### 2. 上传失败
- 检查网络连接
- 验证SSH访问权限
- 确认服务器磁盘空间

#### 3. 服务启动失败
```bash
# 连接服务器检查
ssh root@47.97.185.117
pm2 logs english-learning-backend
```

#### 4. 网站无法访问
```bash
# 检查Nginx状态
nginx -t
systemctl status nginx
```

### 紧急恢复步骤
```bash
# 1. 连接服务器
ssh root@47.97.185.117

# 2. 检查服务状态
pm2 list

# 3. 重启服务
pm2 restart english-learning-backend

# 4. 重启Nginx
systemctl restart nginx

# 5. 检查日志
pm2 logs english-learning-backend --lines 50
```
