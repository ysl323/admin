### 步骤 4: 备份当前数据


### 步骤 5: 数据库迁移

```bash
cd /root/english-learning/my1/backend

# 检查是否需要迁移
sqlite3 database.sqlite ".schema word_mastery" | grep "unique_user_lesson_word_mastery"
```

- **如果没有输出**: 需要执行迁移,继续下面的命令
- **如果有输出**: 索引已是最新,跳到步骤6

**执行数据库迁移:**

```bash
sqlite3 database.sqlite <<EOF
-- 备份现有数据
CREATE TABLE IF NOT EXISTS word_mastery_backup AS SELECT * FROM word_mastery;

-- 删除旧索引
DROP INDEX IF EXISTS unique_user_word_mastery;

-- 创建新索引(包含lessonId)
CREATE UNIQUE INDEX IF NOT EXISTS unique_user_lesson_word_mastery
ON word_mastery (userId, lessonId, wordId);

-- 验证
SELECT '迁移完成' as status;
EOF
```

## ❌ 常见错误处理

### 错误 1: `git pull origin master` 失败

**错误信息:** `fatal: not a git repository`

**解决方案:**
```bash
# 检查是否有.git目录
ls -la

# 如果没有,需要手动上传代码
# 请联系开发人员获取代码包
```

---

### 错误 2: `npm install` 失败

**错误信息:** `npm ERR! code ENOENT` 或 `command not found: npm`

**解决方案:**
```bash
# 检查npm是否安装
npm --version

# 如果未安装,安装Node.js
curl -fsSL https://rpm.nodesource.com/setup_18.x | bash -
yum install -y nodejs
```

---

### 错误 3: PM2服务启动失败

**错误信息:** `pm2: command not found`

**解决方案:**
```bash
# 安装PM2
npm install -g pm2
pm2 startup
pm2 save
```

---

### 错误 4: 前端构建失败

**错误信息:** `Build failed with errors`

**解决方案:**
```bash
cd /root/english-learning/my1/frontend

# 清理缓存重新构建
rm -rf node_modules package-lock.json dist
npm install
npm run build
```

---

### 错误 5: 网站无法访问

**解决方案:**
```bash
# 检查Nginx状态
systemctl status nginx

# 启动Nginx
systemctl start nginx

# 检查防火墙
firewall-cmd --list-all

# 如果需要开放端口
firewall-cmd --zone=public --add-port=80/tcp --permanent
firewall-cmd --reload
```

