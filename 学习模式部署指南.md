# 学习模式功能部署指南

## 功能概述

本次部署包含以下新增功能:
- 小白模式: 未掌握单词显示英文,已掌握隐藏英文
- 进阶模式: 始终隐藏英文(听写模式)
- 掌握状态持久化: 本地存储 + 服务器同步
- 离线数据同步: 支持批量同步离线掌握记录

## 部署前准备

### 1. 数据库迁移

需要更新 `word_mastery` 表的唯一索引,从 `(userId, wordId)` 改为 `(userId, lessonId, wordId)`。

**在服务器上执行以下SQL**:

```sql
-- 备份现有数据(如果需要)
CREATE TABLE word_mastery_backup AS SELECT * FROM word_mastery;

-- 删除旧索引
DROP INDEX IF EXISTS unique_user_word_mastery ON word_mastery;

-- 创建新索引
CREATE UNIQUE INDEX unique_user_lesson_word_mastery
ON word_mastery (userId, lessonId, wordId);
```

## 常见问题

### 问题1: 数据库索引冲突

**症状**: `Duplicate entry` 错误

**解决**:
```sql
-- 查找重复数据
SELECT userId, lessonId, wordId, COUNT(*) as cnt
FROM word_mastery
GROUP BY userId, lessonId, wordId
HAVING cnt > 1;

-- 删除重复记录(保留最新的一条)
DELETE FROM word_mastery
WHERE id NOT IN (
  SELECT MIN(id) FROM word_mastery
  GROUP BY userId, lessonId, wordId
);
```

### 问题2: 掌握状态不同步

**症状**: 本地显示已掌握,刷新后丢失

**解决**:
- 检查 `localStorage` 中是否有 `word_mastery_*` 键
- 检查网络请求是否成功(F12开发者工具)
- 查看后端日志是否有错误

### 问题3: 后端启动失败

**症状**: `pm2 restart` 后服务无法启动

**解决**:
```bash
# 查看详细错误
pm2 logs english-backend --err

# 检查数据库文件权限
ls -la backend/database.sqlite

# 检查环境变量
cat backend/.env
```

## 性能优化建议

1. **数据库索引**: 已添加复合唯一索引 `(userId, lessonId, wordId)`,查询性能已优化

2. **离线同步**: 建议定期执行同步,避免大量离线数据堆积
   ```javascript
   // 在App.vue中添加
   onMounted(() => {
     learningStore.syncOfflineMasteryData();
   });
   ```

3. **缓存策略**: 可考虑对掌握状态进行Redis缓存,减少数据库查询

4. **批量操作**: 获取课程掌握数据时使用 `findAll` 而非多次 `findOrCreate`
