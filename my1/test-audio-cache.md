# 音频缓存功能测试指南

## 前置条件

✅ 数据库表已创建（audio_caches）
✅ 后端服务已重启（端口 3000）
✅ 前端服务运行中（端口 5173）

## 测试步骤

### 1. 测试缓存自动生成

1. 打开浏览器访问：http://localhost:5173
2. 登录用户账号（非管理员账号即可）
3. 进入学习页面，选择任意课程
4. 点击播放一个单词的音频（例如 "Hello"）
5. 查看后端日志，应该看到：
   ```
   info: 缓存未命中: Hello... (key: abc123...)
   info: 调用火山引擎 TTS API...
   info: 音频已缓存: Hello... (size: 12345 bytes)
   ```

6. 再次点击播放同一个单词
7. 查看后端日志，应该看到：
   ```
   info: 缓存命中: Hello... (key: abc123...)
   ```

### 2. 测试缓存管理界面

1. 使用管理员账号登录
   - 用户名：admin
   - 密码：（查看 ADMIN-ACCOUNT-INFO.md）

2. 进入"后台管理" → "缓存管理"
   - URL: http://localhost:5173/admin/cache

3. 验证统计信息显示：
   - 缓存总数
   - 总大小（字节）
   - 总命中次数
   - 平均大小
   - 命中率

4. 验证缓存列表显示：
   - 文本内容
   - 提供商（volcengine）
   - 文件大小
   - 命中次数
   - 创建时间

### 3. 测试缓存操作

#### 3.1 播放测试
1. 在缓存列表中找到任意一条记录
2. 点击"播放"按钮
3. 应该能听到音频播放

#### 3.2 删除单个缓存
1. 点击某条记录的"删除"按钮
2. 确认删除
3. 列表应该更新，该记录消失
4. 统计信息应该更新

#### 3.3 搜索功能
1. 在搜索框输入单词（例如 "Hello"）
2. 点击"搜索"
3. 列表应该只显示包含该单词的缓存

#### 3.4 刷新列表
1. 点击"刷新"按钮
2. 列表应该重新加载

#### 3.5 清空所有缓存
1. 点击"清空所有缓存"按钮
2. 确认操作
3. 所有缓存应该被删除
4. 统计信息归零

### 4. 验证缓存文件

#### 4.1 查看缓存目录
```bash
cd my1/backend
dir audio-cache
```

应该看到 .mp3 文件，文件名是 MD5 哈希值。

#### 4.2 查看数据库
```bash
cd my1/backend
sqlite3 database.sqlite
```

```sql
-- 查看所有缓存
SELECT * FROM audio_caches;

-- 查看统计
SELECT 
  COUNT(*) as total_count,
  SUM(file_size) as total_size,
  SUM(hit_count) as total_hits,
  AVG(file_size) as avg_size
FROM audio_caches;

-- 按提供商统计
SELECT 
  provider,
  COUNT(*) as count,
  SUM(file_size) as size
FROM audio_caches
GROUP BY provider;
```

### 5. 测试缓存命中率

1. 播放 10 个不同的单词（首次播放）
   - 应该看到 10 次"缓存未命中"
   - 应该调用 10 次火山引擎 API

2. 再次播放这 10 个单词
   - 应该看到 10 次"缓存命中"
   - 不应该调用火山引擎 API

3. 查看缓存管理界面
   - 缓存总数：10
   - 总命中次数：10
   - 命中率：1.00（100%）

### 6. 性能对比

#### 首次播放（无缓存）
- 响应时间：500-2000ms
- 需要调用火山引擎 API

#### 再次播放（有缓存）
- 响应时间：< 50ms
- 直接从本地读取
- **速度提升 10-40 倍**

## 故障排查

### 问题 1：后端报错 "no such table: audio_caches"

**解决方案**：
```bash
# 运行数据库同步脚本
sync-database.bat

# 或者重启后端服务
restart-backend.bat
```

### 问题 2：缓存目录不存在

**解决方案**：
```bash
cd my1/backend
mkdir audio-cache
```

后端会自动创建，但如果遇到权限问题，可以手动创建。

### 问题 3：管理界面无法访问

**检查**：
1. 是否使用管理员账号登录
2. 后端服务是否正常运行
3. 路由是否正确注册

### 问题 4：音频无法播放

**检查**：
1. 缓存文件是否存在（audio-cache 目录）
2. 数据库记录是否正确
3. 浏览器控制台是否有错误
