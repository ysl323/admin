# 清空服务器并重新部署 - 使用说明

## 概述

这些脚本用于完全清空服务器上的应用并重新部署本地最新代码，包括所有最新的修复和功能。

## 可用脚本

### 1. PowerShell 版本（推荐）
**文件**: `清空服务器并重新部署.ps1`

**特点**:
- 功能最完整
- 支持参数选项
- 详细的进度显示
- 错误处理完善

**使用方法**:
```powershell
# 基本使用
.\清空服务器并重新部署.ps1

# 跳过前端构建（如果已经构建过）
.\清空服务器并重新部署.ps1 -SkipBuild

# 不保留数据库（使用全新数据库）
.\清空服务器并重新部署.ps1 -KeepDatabase:$false

# 指定不同的服务器IP
.\清空服务器并重新部署.ps1 -ServerIP "your.server.ip"
```

### 2. 批处理版本
**文件**: `清空服务器并重新部署.bat`

**特点**:
- 兼容性好
- 操作简单
- 适合Windows用户

**使用方法**:
```cmd
清空服务器并重新部署.bat
```

## 部署流程

### 本地操作
1. **构建前端** - 清理并重新构建前端生产版本
2. **创建部署包** - 打包所有必要文件
3. **压缩文件** - 创建ZIP压缩包
4. **上传到服务器** - 通过SCP上传

### 服务器操作
1. **停止服务** - 停止所有相关的PM2服务
2. **备份数据库** - 备份现有数据库（可选）
3. **清空目录** - 完全删除旧的应用文件
4. **创建结构** - 重新创建目录结构
5. **解压文件** - 解压并整理新文件
6. **恢复数据库** - 恢复数据库（如果选择保留）
7. **安装依赖** - 安装Node.js依赖
8. **更新配置** - 更新Nginx配置
9. **启动服务** - 启动新的应用服务

## 故障排除

### 常见问题

**1. 前端构建失败**
```bash
# 解决方案：清理并重新安装依赖
cd frontend
rm -rf node_modules package-lock.json
npm install
npm run build
```

**2. 上传失败**
- 检查网络连接
- 确认服务器SSH访问权限
- 验证服务器磁盘空间

**3. 服务启动失败**
```bash
# 在服务器上检查日志
pm2 logs english-learning-backend
```

**4. 数据库问题**
```bash
# 检查数据库文件权限
ls -la /root/english-learning/backend/database.sqlite
```

### 手动恢复

如果自动部署失败，可以手动执行：

```bash
# 连接到服务器
ssh root@47.97.185.117

# 检查服务状态
pm2 list

# 重启服务
pm2 restart english-learning-backend

# 检查Nginx状态
nginx -t
systemctl status nginx
```
