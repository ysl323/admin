# 🔧 验证码失败修复指南

## 问题现象

注册页面显示：**"获取验证码失败"**

## 快速诊断

### 步骤1: 运行诊断脚本

```bash
cd my1
.\diagnose-captcha.bat
```

### 步骤2: 检查服务器状态

确认两个服务器都在运行：

1. **后端服务器** (端口 3000)
   ```bash
   curl http://localhost:3000/api/captcha
   ```
   应该返回类似：
   ```json
   {
     "success": true,
     "captchaId": "...",
     "question": "5 + 3 = ?"
   }
   ```

2. **前端服务器** (端口 5173)
   ```bash
   curl http://localhost:5173
   ```
   应该返回 HTML 内容

## 常见原因和解决方案

### 原因1: 后端服务器未运行 ⭐ 最常见

**症状**: 
- 浏览器控制台显示 `ERR_CONNECTION_REFUSED`
- Network 标签显示请求失败

**解决方案**:
```bash
cd my1
.\start-all.bat
```

或单独启动后端：
```bash
cd my1\backend
npm start
```

### 原因2: 前端服务器未运行

**症状**:
- 无法访问 http://localhost:5173

**解决方案**:
```bash
cd my1\frontend
npm run dev
```

### 原因3: CORS 错误

**症状**:
- 浏览器控制台显示 CORS 相关错误
- 类似：`Access to XMLHttpRequest has been blocked by CORS policy`

**解决方案**:

检查 `backend/.env` 文件：
```env
CORS_ORIGIN=http://localhost:5173
```

如果没有这行，添加它并重启后端。

### 原因4: 端口冲突

**症状**:
- 服务器启动失败
- 显示 `EADDRINUSE` 错误

**解决方案**:

1. 关闭占用端口的进程：
   ```bash
   # 查找占用 3000 端口的进程
   netstat -ano | findstr :3000
   
   # 杀死进程 (替换 PID)
   taskkill /F /PID <PID>
   ```

2. 或修改端口（在 `backend/.env`）：
   ```env
   PORT=3001
   ```

### 原因5: API 路径错误

**症状**:
- Network 标签显示 404 错误
- 请求的 URL 不正确

**检查**:
- 前端应该请求: `http://localhost:3000/api/captcha`
- 后端路由应该是: `/api/captcha`

## 详细调试步骤

### 1. 打开浏览器开发者工具

按 `F12` 打开开发者工具

### 2. 查看 Console 标签页

查找红色错误信息：
- `ERR_CONNECTION_REFUSED` → 后端未运行
- `CORS error` → CORS 配置问题
- `404 Not Found` → API 路径错误
- `Network Error` → 网络连接问题

### 3. 查看 Network 标签页

1. 刷新页面
2. 找到 `captcha` 请求
3. 查看请求详情：
   - **Status**: 应该是 200
   - **Response**: 应该有 JSON 数据
   - **Headers**: 检查 CORS 头

### 4. 测试后端 API

在浏览器中直接访问：
```
http://localhost:3000/api/captcha
```

应该看到 JSON 响应：
```json
{
  "success": true,
  "captchaId": "abc123...",
  "question": "7 + 9 = ?"
}
```

## 完整重启流程

如果以上都不行，尝试完全重启：

### 1. 停止所有服务

```bash
cd my1
.\stop-all.bat
```

或手动：
```bash
# 杀死所有 node 进程
taskkill /F /IM node.exe
```

### 2. 清理并重新安装依赖（可选）

```bash
# 后端
cd my1\backend
rmdir /s /q node_modules
npm install

# 前端
cd my1\frontend
rmdir /s /q node_modules
npm install
```

### 3. 重新启动

```bash
cd my1
.\start-all.bat
```

### 4. 等待服务器启动

- 后端: 等待 "服务器运行在端口 3000" 消息
- 前端: 等待 "Local: http://localhost:5173" 消息

### 5. 测试

访问: http://localhost:5173/register
