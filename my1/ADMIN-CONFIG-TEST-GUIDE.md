## 技术实现细节

### 后端实现

1. **加密存储**
   - 使用 AES-256-CBC 算法加密敏感配置
   - 加密密钥从环境变量 `ENCRYPTION_KEY` 读取
   - 加密格式：`iv:encryptedData`

2. **脱敏显示**
   - API Key 和 Secret 只显示前4位和后4位
   - 格式：`sRWj****NPL`

3. **数据库模型**
   - Config 表存储所有配置
   - 使用 key-value 结构
   - 支持 upsert 操作

4. **级联删除**
   - 删除分类时自动删除所有课程和单词
   - 删除课程时自动删除所有单词
   - 使用数据库事务确保数据一致性

### 前端实现

1. **Element Plus 组件**
   - 使用 `el-tabs` 实现标签页切换
   - 使用 `el-table` 显示数据列表
   - 使用 `el-dialog` 实现弹窗表单
   - 使用 `el-message-box` 实现二次确认

2. **按钮类型修复**
   - 将所有 `size="small"` 的按钮改为 `link` 类型
   - 消除 Element Plus 警告

3. **表单验证**
   - 使用 `el-form` 的 `rules` 属性
   - 必填项验证
   - URL 格式验证

## 故障排查

### 问题1：404 错误

**症状：** 访问 `/api/admin/categories` 等接口返回 404

**解决方案：**
- 确认后端服务已重启
- 检查 `backend/src/routes/admin.js` 是否包含 GET 路由
- 检查路由顺序（GET 路由应在 POST 路由之前）

### 问题2：配置保存失败

**症状：** 保存 TTS 配置时提示失败

**解决方案：**
- 检查 `.env` 文件是否包含 `ENCRYPTION_KEY`
- 检查数据库 Config 表是否存在
- 查看后端日志获取详细错误信息

### 问题3：密钥未脱敏

**症状：** 配置页面显示完整密钥

**解决方案：**
- 检查后端 `/api/admin/config/tts` GET 路由是否包含脱敏逻辑
- 确认 `maskSensitive` 函数正常工作

### 问题4：Element Plus 警告

**症状：** 控制台显示按钮类型警告

**解决方案：**
- 将 `<el-button size="small">` 改为 `<el-button link size="small">`
- 或者移除 `size="small"` 属性
