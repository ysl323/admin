## 发现的问题

### 1. 音频播放冲突错误
**错误信息：**
```
AudioManager.js:60 音频播放失败: DOMException: The play() request was interrupted by a call to pause()
```

**原因：**
在 `AudioManager.play()` 方法中，先调用 `this.stop()` 会立即执行 `pause()`，然后马上调用 `play()` 导致冲突。

**解决方案：**
- 修改 `AudioManager.js` 的 `play()` 方法
- 在停止当前播放后，添加 50ms 延迟确保停止完成
- 只在正在播放时才调用 `stop()`

**修改文件：**
- `my1/frontend/src/utils/AudioManager.js`

### 2. API 500 错误 - /api/admin/words
**错误信息：**
```
/api/admin/words:1 Failed to load resource: the server responded with a status of 500 (Internal Server Error)
```

**原因：**
在 `AdminService.getAllWords()` 方法中，访问 `word.lesson.category.name` 时，如果数据库中存在没有关联课程或分类的单词，会导致空指针错误。

**解决方案：**
- 修改 `AdminService.getAllWords()` 方法
- 在 Sequelize 查询中添加 `required: false` 允许左连接
- 使用可选链操作符 `?.` 安全访问嵌套属性
- 添加空值检查和默认值

**修改文件：**
- `my1/backend/src/services/AdminService.js`

**修改内容：**
```javascript
// 添加 required: false 允许左连接
include: [
  {
    model: Lesson,
    as: 'lesson',
    required: false,  // 新增
    // ...
  }
]

// 安全访问嵌套属性
const lesson = word.lesson;
const category = lesson?.category;

return {
  // ...
  lessonInfo: (lesson && category)
    ? `${category.name} - 第${lesson.lessonNumber}课`
    : '未知课程',
  categoryId: lesson?.categoryId || null,
  categoryName: category?.name || '未知分类',
  lessonNumber: lesson?.lessonNumber || null
};
```

### 3. Element Plus API 警告
**警告信息：**
```
[el-radio] [API] label act as value is about to be deprecated in version 3.0.0, please use value instead.
```

**原因：**
Element Plus 3.0 版本中，`el-radio` 组件的 `label` 属性已被弃用，应该使用 `value` 属性。

**解决方案：**
将所有 `el-radio` 组件的 `label` 属性改为 `value` 属性。

**修改文件：**
1. `my1/frontend/src/views/admin/ContentManagement.vue`
   - 第 190-191 行：导入方式选择
   
2. `my1/frontend/src/views/admin/ConfigManagement.vue`
   - 第 26-27 行：配置模式选择

**修改示例：**
```vue
<!-- 修改前 -->
<el-radio label="text">粘贴 JSON 数据</el-radio>
<el-radio label="file">上传 JSON 文件</el-radio>

<!-- 修改后 -->
<el-radio value="text">粘贴 JSON 数据</el-radio>
<el-radio value="file">上传 JSON 文件</el-radio>
```

## 数据库问题说明

如果你看到"未知课程"或"未知分类"，可能是因为：
- 课程关联到了不存在的分类（分类被删除了）
- 单词关联到了不存在的课程（课程被删除了）

这不会导致 500 错误（因为我们已经修复了），但会显示"未知"。

### 解决方案
1. 在管理后台删除这些孤立的数据
2. 或者重新导入完整的数据
