## 立即开始使用

### 方式 1：在学习页面自动使用（推荐）

1. 打开浏览器：http://localhost:5173
2. 登录任意用户账号
3. 进入学习页面，播放单词音频
4. 系统会自动：
   - 首次播放：调用火山引擎 API → 缓存音频
   - 再次播放：直接使用缓存（速度快 10-40 倍）

### 方式 2：管理缓存

1. 使用管理员账号登录
2. 进入"后台管理" → "缓存管理"
3. 查看统计信息和缓存列表
4. 可以播放、删除、搜索缓存

### 方式 3：运行自动化测试

```bash
# 在项目根目录运行
test-audio-cache.bat
```

这会自动测试所有 API 功能并显示性能对比。

## 核心优势

### 成本节省
- 重复单词不再调用 API
- 预计节省 70-90% 的 API 调用成本

### 性能提升
- 缓存命中响应时间：< 50ms
- API 调用响应时间：500-2000ms
- 速度提升 10-40 倍

### 用户体验
- 音频加载更快
- 学习更流畅
- 无需等待

## 工作原理

```
用户请求音频
    ↓
查询本地缓存
    ↓
缓存存在？
├─ 是 → 返回缓存音频 ⚡ (< 50ms)
└─ 否 → 调用火山引擎 API
        保存到缓存
        返回音频 ✅
```

## 常见问题

### Q: 如何知道缓存是否生效？

A: 查看后端日志，会显示"缓存命中"或"缓存未命中"。

### Q: 缓存文件存储在哪里？

A: `backend/audio-cache/` 目录，文件名是 MD5 哈希值。

### Q: 如何清空所有缓存？

A: 在管理界面点击"清空所有缓存"按钮，或者手动删除 `backend/audio-cache/` 目录中的文件。

### Q: 缓存会占用多少空间？

A: 每个音频约 10-100KB，1000 个单词约 10-100MB。

### Q: 如果遇到数据库错误怎么办？

A: 运行 `sync-database.bat` 重新同步数据库。
