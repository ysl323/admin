核心功能测试

测试用例 1: 输入框宽度动态变化

测试步骤：
  1. 找到一个多单词题目（如 "look at"）
  2. 在第一个输入框中，逐个字符输入 "l"
  3. 观察输入框宽度是否变化
  4. 继续输入 "o", "o", "k"
  5. 每输入一个字符，观察输入框宽度
预期结果：
  • ✅ 输入框宽度应该随着输入字符数量实时增长
  • ✅ 输入1个字符时，宽度应该比初始状态宽
  • ✅ 输入4个字符时，宽度应该比输入1个字符时更宽
  • ❌ 如果宽度始终不变，说明BUG未修复

测试用例 2: Placeholder下划线动态减少

测试步骤：
  1. 在第一个输入框中，观察初始placeholder（应该是4个下划线 "____"）
  2. 输入 "l"，观察placeholder变化
  3. 输入 "o"，观察placeholder变化
  4. 输入 "o"，观察placeholder变化
  5. 输入 "k"，观察placeholder变化
预期结果：
  • ✅ 初始状态：placeholder显示 "____" (4个下划线)
  • ✅ 输入 "l" 后：placeholder显示 "___" (3个下划线)
  • ✅ 输入 "lo" 后：placeholder显示 "__" (2个下划线)
  • ✅ 输入 "loo" 后：placeholder显示 "_" (1个下划线)
  • ✅ 输入 "look" 后：placeholder显示 "" (无下划线)
  • ❌ 如果下划线数量不变，说明BUG未修复

测试用例 3: 错误时只清空错误的单词

测试步骤：
  1. 在第一个输入框输入正确的 "look"
  2. 按空格键切换到第二个输入框
  3. 在第二个输入框输入错误的 "xx"
  4. 按空格键提交
  5. 观察哪些输入框被清空
预期结果：
  • ✅ 第一个输入框（"look"）应该保留内容
  • ✅ 第二个输入框（"xx"）应该被清空
  • ✅ 第二个输入框应该显示红色错误状态
  • ✅ 焦点应该自动回到第二个输入框
  • ❌ 如果两个输入框都被清空，说明BUG未修复

测试用例 4: 键盘敲击声音

测试步骤：
  1. 确保浏览器音量已打开
  2. 在输入框中输入任意字符

预期结果：
  • ✅ 每次输入字符时应该听到短促的"哒"声
  • ✅ 声音应该清脆、不刺耳

测试用例 5: 空格键切换输入框

测试步骤：
  1. 在第一个输入框输入 "look"
  2. 按空格键
  3. 观察焦点是否切换到第二个输入框

  • ✅ 焦点应该自动切换到第二个输入框
