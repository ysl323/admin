## 🎯 三种部署方案

### 方案1: 本地测试（当前状态）

**优点**: 快速、简单
**适合**: 开发和测试

1. 启动本地服务
   ```bash
   # 后端（端口3000）
   cd e:\demo\my1\my1\my1\backend
   npm run dev

   # 前端（端口5173）
   cd e:\demo\my1\my1\my1\frontend
   npm run dev
   ```

2. 访问 http://localhost:5173
3. 使用 `admin` / `admin123` 登录

### 方案2: 推送到GitHub（推荐）

**优点**: 代码备份、版本控制
**适合**: 团队协作、版本管理

**步骤**:

1. **安装Git**（如果还没有）
   - 下载: https://git-scm.com/downloads
   - 安装并重启命令行

2. **推送到GitHub**
   ```bash
   # 方式1: 使用脚本（推荐）
   双击运行: push-to-github.bat

   # 方式2: 手动执行
   cd e:\demo\my1\my1\my1
   git init
   git add .
   git commit -m "初始化项目"
   git remote add origin https://github.com/你的用户名/你的仓库.git
   git push -u origin master
   ```

3. **在GitHub上查看**
   - 访问你的GitHub仓库
   - 查看所有代码和文档

### 方案3: 部署到服务器（生产环境）

**优点**: 在线访问、稳定运行
**适合**: 生产环境、正式上线

**步骤A: 准备GitHub仓库**

1. 先完成方案2，推送到GitHub
2. 确保GitHub仓库URL正确

**步骤B: 部署到服务器**

```bash
# 1. 上传更新脚本到服务器
scp update-server-fix-login.sh root@47.97.185.117:/root/

# 2. SSH登录服务器
ssh root@47.97.185.117

# 3. 执行更新脚本
cd /root
chmod +x update-server-fix-login.sh
./update-server-fix-login.sh
```

或者手动执行：

```bash
# SSH登录服务器
ssh root@47.97.185.117

# 进入项目目录
cd /www/wwwroot/english-learning/backend

# 执行修复
node fix-user-table.js
node reset-admin-password.js

# 重启服务
pm2 restart english-backend
```

**步骤C: 验证部署**

- 访问: http://47.97.185.117
- 使用 `admin` / `admin123` 登录
