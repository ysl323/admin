# 快速部署 - 学习模式功能

## 部署文件说明

### 1. deploy-learning-mode.sh
自动化部署脚本,包含以下步骤:
- 备份当前代码和数据库
- Git拉取最新代码
- 数据库索引迁移
- 后端依赖安装
- 前端构建
- 服务重启

### 2. migrate-word-mastery.sql
数据库迁移脚本,手动执行或通过部署脚本自动执行。

### 3. 学习模式部署指南.md
完整部署文档,包含详细说明和故障排查。

---

## 推荐部署方式

### 方式一: 使用自动化脚本(最简单)

在服务器上执行:

```bash
# 1. 上传脚本到服务器
scp deploy-learning-mode.sh root@47.97.185.117:/root/

# 2. SSH连接到服务器
ssh root@47.97.185.117

# 3. 执行部署脚本
chmod +x deploy-learning-mode.sh
./deploy-learning-mode.sh
```

脚本会自动完成所有部署步骤,并在最后显示部署状态。

---

### 方式二: 手动部署(更多控制)

```bash
# 1. 连接服务器
ssh root@47.97.185.117

# 2. 进入项目目录
cd /root/english-learning/my1

# 3. 停止服务
pm2 stop english-backend

# 4. 更新代码(Git方式)
git pull origin master
# 或手动替换文件

# 5. 数据库迁移
cd backend
sqlite3 database.sqlite < /path/to/migrate-word-mastery.sql

# 6. 安装后端依赖
npm install --production

# 7. 初始化数据库
npm run db:init

# 8. 构建前端
cd ../frontend
npm install
npm run build

# 9. 重启服务
pm2 restart english-backend

# 10. 检查状态
pm2 status
pm2 logs english-backend --lines 50
```

## 常见问题处理

### 问题1: PM2启动失败

```bash
# 查看错误日志
pm2 logs english-backend --err

# 常见原因:
# - Node.js版本不兼容 → 更新Node.js
# - 数据库权限问题 → chmod 644 database.sqlite
# - 端口被占用 → lsof -i:3000
```

### 问题2: 数据库索引错误

```bash
# 查看错误
sqlite3 database.sqlite ".schema word_mastery"

# 手动执行迁移
sqlite3 database.sqlite < migrate-word-mastery.sql
```

### 问题3: 前端构建失败

```bash
# 清理缓存后重试
cd frontend
rm -rf node_modules dist
npm install
npm run build
```

### 问题4: 掌握状态不显示

1. 打开浏览器F12开发者工具
2. 切换到Console标签
3. 查看是否有错误信息
4. 切换到Network标签
5. 查看API请求是否成功

## 日志位置

### 后端日志
- PM2标准输出: `~/.pm2/logs/english-backend-out.log`
- PM2错误输出: `~/.pm2/logs/english-backend-error.log`
- 应用日志(如果配置): `backend/logs/*.log`

### Nginx日志
- 访问日志: `/var/log/nginx/access.log`
- 错误日志: `/var/log/nginx/error.log`

### 实时查看日志

```bash
# PM2日志(实时)
pm2 logs english-backend

# 最后100行错误日志
pm2 logs english-backend --lines 100 --err

# Nginx访问日志
tail -f /var/log/nginx/access.log
```

## 回滚方案

如果部署后出现问题,可以快速回滚:

```bash
# 1. 停止服务
pm2 stop english-backend

# 2. 回滚代码
cd /root/english-learning/my1
git log              # 查看提交历史
git reset --hard <commit-id>  # 回滚到指定提交

# 3. 回滚数据库(如需要)
cp database.sqlite.backup database.sqlite

# 4. 重启服务
pm2 restart english-backend

# 5. 验证
pm2 status
```
