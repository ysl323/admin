# 导出缓存 500 错误 - 快速修复

## 问题
点击"导出缓存"按钮时返回 500 错误。

## 最快的解决方案

### 方法1: 使用自动修复脚本（推荐）
```bash
fix-export-500.bat
```

这个脚本会：
1. 检查后端服务状态
2. 测试导出功能
3. 如果需要，自动重启后端

### 方法2: 手动重启后端
1. **停止后端**:
   - 找到运行后端的命令行窗口
   - 按 `Ctrl+C`
   - 等待完全停止

2. **启动后端**:
   ```bash
   cd my1\backend
   npm start
   ```

3. **等待启动完成**，看到：
   - "数据库连接成功"
   - "数据库同步完成"
   - "服务器运行在端口 3000"

4. **测试**:
   - 刷新浏览器页面
   - 重新登录（如果需要）
   - 点击"导出缓存"

## 为什么会出现这个问题？

最可能的原因是：**后端代码修改后没有真正重启**

即使你之前重启过，可能：
- 进程没有完全停止
- 旧代码仍在运行
- 新代码没有加载

## 验证修复

修复后，测试以下内容：

1. **健康检查**:
   - 访问: http://localhost:3000/health
   - 应该看到: `{"status":"ok","timestamp":"..."}`

2. **导出功能**:
   - 登录管理员账号
   - 进入缓存管理页面
   - 点击"导出缓存"
   - 应该自动下载 JSON 文件
