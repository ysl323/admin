# 音频播放切换问题修复

## 问题描述
用户在学习页面快速切换题目时，上一题的音频还在播放，导致音频重叠。

## 问题原因
在切换题目（上一题/下一题）或答对题目自动跳转时，没有先停止当前正在播放的音频，就直接开始播放新题目的音频。

## 修复方案

### 1. AudioManager.js
- 修改 `play()` 方法：在播放新音频前自动调用 `stop()` 停止当前音频
- 移除了"音频正在播放中"的警告，改为直接停止并播放新音频

### 2. LearningPage.vue
在以下场景添加音频停止逻辑：

1. **playAudio 函数**：
   - 在函数开始时先调用 `AudioManager.stop()` 停止当前音频

2. **handlePrevious 函数**（上一题）：
   - 在切换题目前先停止音频
   - 重置 `isPlaying` 状态

3. **handleNext 函数**（下一题）：
   - 在切换题目前先停止音频
   - 重置 `isPlaying` 状态

4. **handleSubmit 函数**（答对自动跳转）：
   - 在跳转到下一题前先停止音频
   - 重置 `isPlaying` 状态

## 修复效果
- 切换题目时，上一题的音频会立即停止
- 新题目的音频会立即开始播放
- 不会出现音频重叠的情况
- 响应速度更快，用户体验更好

## 测试方法
1. 打开学习页面
2. 开始学习任意课程
3. 在音频播放过程中快速点击"上一题"或"下一题"
4. 验证：
   - 上一题的音频立即停止
   - 新题目的音频立即开始播放
   - 没有音频重叠
