## 问题根本原因
数据库中存在8个课程，其中3个课程（ID: 1, 2, 3）的分类ID指向了不存在的分类（ID=5），导致这些课程无法在前端显示。

## 修复内容

### 修复前数据状态
- **分类**: 2个（新概念3、esp32-idf）
- **课程**: 8个（其中3个无法显示）
- **单词**: 106个

### 修复操作
使用SQL直接更新无效的课程分类关联：
```sql
UPDATE lessons SET category_id = 10, lesson_number = ? WHERE id = ?
```

具体修复：
- 课程1 (原第1课) -> 改为第5课，分类ID改为10（新概念3）
- 课程2 (原第2课) -> 改为第6课，分类ID改为10（新概念3）
- 课程3 (原第3课) -> 改为第7课，分类ID改为10（新概念3）

### 修复后数据状态

- 第1课（课程ID: 4）
